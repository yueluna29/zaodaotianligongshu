---
name: 不加多余的"新/预览/临时"类标签
description: 菜单/按钮/标题用最终态命名，不要为了"标记这是新的"或"以防混淆"加 (新)、(preview)、(beta)、(temp) 之类后缀
type: feedback
---

加新菜单/按钮/页面标题时，直接用最终命名，不要加 `(新)`、`(预览)`、`(beta)`、`(临时)`、`(WIP)` 这种"过程性"后缀。

**Why**：这是用户在线上看的产品 UI，加这种括号注释是从开发视角出发，对用户没价值，反而让界面显得未完成。即便和已有功能名字撞了，也应该想办法重命名其中一个，而不是加括号区分。

**How to apply**：
- 加新菜单项时直接用目标名（哪怕和老菜单暂时重名要等切换，也要规划好命名而不是加括号）
- 不在共享 UI 里暴露内部状态（"这是新版"不是用户关心的事）
- 当前案例：新建 `LeaveHub` 菜单时给的是 `假期管理 (新)`，应该直接是 `假期管理`
