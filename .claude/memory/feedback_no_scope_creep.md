---
name: 严格按用户指令的范围改，不要顺手改别的
description: 用户说改 X 就只改 X。不要"既然在这了顺便也改 Y"，特别是用户没明说的另一个端/场景/视图
type: feedback
---

用户让改某个具体场景（比如「手机版」），就**只改那个场景**，不要顺手把其它场景（桌面版）也"统一优化了"。

**Why**：用户对每个端的视觉/布局有自己的偏好，桌面版当前样子可能就是她想要的。我自作主张统一改，会破坏她已经认可的部分，触发"还不如原来的"反馈和返工。她的原话："你到底什么时候才能不要自作主张？我让你改手机版，你干嘛在桌面版也改了？"

**How to apply**：
- 听到"手机版/移动端/PC/桌面/打印/某 tab/某页面"等具体作用域 → 用条件渲染只改那一处
- 不要把"既然在这了顺便也整理一下"作为加戏理由
- 改之前没把握就问："只改手机还是桌面也一起改？"
- 真觉得另一个场景也有改进空间 → 改完原任务后**口头建议**，由用户决定，不直接动手
- 当前案例：用户说手机版 tab 排列难看，我把桌面版也拆成两行 → 错。正确做法是窗口宽度判断，桌面保持原样
