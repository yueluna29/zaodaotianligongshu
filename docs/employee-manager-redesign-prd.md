# 人事档案（EmployeeManager）UI 重设计 PRD

## 1. 产品背景

- 系统：早稻田理工塾内部勤怠/工资管理系统
- 技术栈：React 18（函数组件 hooks，**无 TypeScript**）+ Vite + Supabase
- 样式约束：**全部使用内联 style 对象**，通过 theme 对象 `t` 取色；**禁止引入 Tailwind / CSS modules / styled-components 等 CSS 库**
- 图标库：`lucide-react`
- 页面定位：员工档案的总览与编辑中心
- 两种使用者：
  - **管理者（admin）**：能看到所有员工；先进"档案库列表"选人，再进"档案详情"查看/编辑
  - **普通员工（staff）**：直接进入自己的档案详情；看不到列表页，也看不到别人
- 两家关联公司（共享系统）：
  - 日本公司：世家学舍 / 紫陽花教育（显示日本相关字段）
  - 中国公司：早稻大连 / 早理金华（字段更精简）

## 2. 设计主题（theme 对象 `t`）

所有颜色**必须**从 `t` 取，支持深浅两套主题：

| key | 含义 |
|---|---|
| `t.ac` | 主强调色 |
| `t.tx` | 主文字 |
| `t.ts` | 次要文字 |
| `t.tm` | 弱化文字 |
| `t.td` | disabled 文字 |
| `t.bgC` | 卡片底色 |
| `t.bgI` | 输入框底色 |
| `t.bgH` | 表头/悬停底色 |
| `t.bd` | 主边框 |
| `t.bl` | 浅边框/内嵌区块底色 |
| `t.gn` / `t.rd` / `t.wn` | 绿 / 红 / 黄（状态色） |
| `t.we` | 周末底色 |

## 3. 雇佣类型与公司枚举

### 雇佣类型
- **日本公司**：正社員 / 契約社員 / アルバイト / 外部講師
- **中国公司**：正社员 / 兼职
- 工具函数：
  - `isFullTime(type)` → 正社員 / 契約社員 / 正社员
  - `isHourly(type)` → アルバイト / 外部講師 / 兼职

### 部门（根据雇佣类型变化）
- 正/契（full-time）：教务 / 咨询 / 宣传 / 财务
- バイト/外部/兼职（hourly）：大学院 / 学部 / 文书 / 语言类

### 其他枚举
- 地区（仅正/契且为日本公司）：日本 / 中国
- 担任科目（多选，共 17 项）：物理、数学、机械工学、电气电子、情报科学、土木建筑、生命理工、材料化学、环境工学、体育学、大学院文科、经营工学、EJU数学、EJU理科、日语、英语、班主任
- 性别：男 / 女
- 支付方式：银行转账 / 现金
- 交通费方式：实报实销 / 固定
- 账户类型：普通 / 当座

## 4. 页面结构（两级）

### 4.1 档案库列表（仅 admin）

顶部：标题"人事档案库 (N 名)" + "新增社员"按钮

筛选区：
- 公司筛选：4 家公司横向 chip（点击切换）
- 雇佣类型筛选："全部 / 正社員 / 契約社員 / アルバイト / 外部講師 / 正社员 / 兼职"

表格列：
| 姓名（+ 假名副行） | 部门（地区+部门 chip） | 科目（最多 3 个 chip，多的 +N） | 工作类型 | 在留状态（带期限预警） | 操作（"查看档案"） |

点击行进入档案详情。

### 4.2 档案详情

顶部操作栏：
- `← 返回列表`（admin 才有，staff 看不到）
- 标题"人事档案"或"新增社员"
- `编辑档案`按钮（admin 任意员工；staff 只能自己）
- 编辑中时显示：取消 / 保存

顶部员工名片（查看态）：
- 左：圆形头像（首字母 + 主色渐变）
- 中：姓名（粗大）、假名・拼音、登录 ID（monospace 小徽章）、5~6 个 chip（公司色黄 / 雇佣类型色绿 / 地区色蓝 / 部门色主色 / 管理者色紫 / 兼任教师色灰）
- 右：入职日期 + 在职 X 年、联系方式（电话+邮箱）
- 底部警告条（红/黄）：在留卡 90 天内过期 / 合同 60 天内到期

## 5. 字段与权限矩阵

### 区块 1：归属与状态（4 列 grid）

| 字段 | 编辑权限 |
|---|---|
| 所属公司（切换公司会自动修正雇佣类型下拉） | **仅 admin** |
| 工作类型 | **仅 admin** |
| 入职日期 | **仅 admin** |
| 权限（社员/管理者） | **仅 admin** |

staff 只能查看，不能改。

### 区块 2：基本信息

条件渲染：**中国公司 vs 日本公司字段不同**。两种都可 staff 自己编辑。

**日本公司版**：
- 第一排（4 列）：汉字姓名 `*` / 假名(Furigana) / 拼音 / 电话
- 第二排：邮箱 `*`（跨 2~3 格） / 地区（仅 full-time 显示）/ 兼任教师 checkbox
- 负责部门（chip 单选，列表根据雇佣类型切换）
- 担任科目（chip 多选，圆角 pill）
- 住址（3 份宽）+ 邮编（1 份宽）
- 备注（整行）

**中国公司版**：
- 第一排：姓名 `*` / 拼音 / 电话 / 邮箱 `*`
- 第二排：身份证号码 / 出生年月日 / 性别 / 兼任教师 checkbox
- 负责部门（chip）
- 担任科目（chip 多选）
- 住址（整行）
- 备注（整行）

### 区块 3：外国人雇佣状况

**仅日本公司显示**，中国公司整个区块隐藏。staff 可编辑。

- 第一排：在留资格 / 在留卡号码 / 在留期限 / 国籍/地域
- 第二排：出生年月日 / 性别 / 资格外许可（checkbox）/ 空格

查看态若 `residence_expiry` 在 90 天内，红色加粗显示"(即将过期)"。

### 区块 4：薪资与税务

**字段级权限非常细**，分两种情况：

**正/契（full-time）**：
| 字段 | 编辑权限 |
|---|---|
| 提成率 % | **仅 admin** |
| 固定加班 h | **仅 admin** |
| 支付方式 | **仅 admin** |
| 交通费方式（实报/固定） | **仅 admin** |
| 交通费金额（円） | **仅 admin** |
| 交通费上限（円） | **仅 admin** |
| 扶养人数 | **仅 admin** |
| My Number | **仅 admin** |
| 合同开始日/结束日 | **仅 admin** |
| 扶养控除（checkbox） | **仅 admin** |
| 签单提成（checkbox） | **仅 admin** |

**バイト/外部/兼职（hourly）**：
| 字段 | 编辑权限 |
|---|---|
| 提成率/固定加班/交通费相关/扶养控除 | **不显示**（不适用） |
| 支付方式 | **staff 可编辑** |
| 扶养人数 | **staff 可编辑** |
| My Number | **staff 可编辑** |
| 合同开始/结束 | **staff 可编辑** |
| 签单提成（checkbox） | **仅 admin** |

视觉上：admin-only 字段对 staff 应明确表示"锁定"，而不是模糊 disabled。建议有锁图标或专门的视觉状态。

### 区块 5：银行信息

staff 可编辑。**中国 vs 日本字段不同**：

**日本公司**：
- 银行名称 / 支店名 / 账户类型（普通/当座） / 账号
- 户名（カナ，单独一行，半角片假名）

**中国公司**：
- 开户名 / 开户银行 / 账户号码（3 列）

### 区块 6：排班设定

**仅 admin 可编辑**，staff 只查看。

按周一到周日（0~6）配置：
- 每一天：开启/休息 + 若开启则 start_time / end_time
- 查看态：7 个方格横排，每格显示星期+时间（有则主色，无则灰）
- 编辑态：竖排 7 行，点星期按钮切换启用，时间用 `<input type="time">`

数据存储在 `work_schedules` 表（非 employees 表），保存时先全删再全插。

### 区块 7：时薪配置（PayRateSection 组件）

**仅 hourly 类型显示**（バイト / 外部 / 兼职），正社員/正社员隐藏。

由独立组件 `PayRateSection` 处理，包含：
- 多种业务类型的时薪（事務 / 講師（大課）/ 講師（一対一）/ 答疑做題 / 研究計画書修改 / 自定义）
- 每种类型的时薪变更历史
- 涨薪（新增一条带生效日的记录）
- 编辑/删除（仅 admin）

本次重设计**不动这个子组件的内部**，但希望它嵌入新设计时视觉上和上下一致。

### 区块 8：假期余额

**仅正/契（full-time）显示**，hourly 隐藏。staff 和 admin 都能看。

3 个指标卡横排：
- 有休余额（主色）：balance 天 + "本年X + 繰越Y - 已用Z"
- 代休余额（紫）：compUnused 天 + "累计X次换休"
- 即将过期代休（红/绿）：14 天内到期数量

若 `hire_date` 为空，加一个警告卡"未设定入职日期"。

## 8. 现有痛点（请重设计解决）

1. **信息太长，一屏看不完**：8 个区块连续铺开，没有 tab / 折叠 / 锚点导航，员工看自己的档案也要滚很久
2. **字段密度不均**：4 列 grid 在宽屏显得留白多，窄屏又挤，响应式不够优雅
3. **admin-only 字段的"锁定"反馈弱**：现在用 `opacity: 0.5` + `cursor: not-allowed`，staff 很难一眼分辨"哪些字段只有 admin 能改"，希望有更清晰的锁定视觉（如锁图标、灰底、提示文案）
4. **顶部名片 chip 太多**：公司 + 雇佣类型 + 地区 + 部门 + 权限 + 教师最多可能 6 个 chip 挤一行，视觉杂乱
5. **列表页是传统表格**：在移动端体验差；有 6 列且科目 chip 会挤；admin 快速浏览时信息密度不均
6. **两套筛选 chip（公司 + 雇佣类型）视觉上未分层**：现在两组用一条小竖线分隔，不够清晰
7. **区块标题装饰单调**：目前用"左侧 3px 主色竖条 + 标题 + 底边框线"，有点呆板
8. **警告条融入差**：在留过期/合同到期警告现在塞在名片下方，管理员批量看列表时无感知
9. **编辑态和查看态切换不连贯**：没有行内编辑，只能整张表单进入 editing 模式，改一个字段也要整表提交
10. **staff 看自己档案的体验**：应该感觉像"我的个人中心"而不是"管理后台的一条记录"，希望有明显不同的氛围

## 9. 必须保留的约束

- React + **内联 style** + lucide-react，**不能引入 CSS 库**
- 颜色必须从 `t.*` 取，支持明暗主题
- 所有文案中文；日文仅用于系统术语（正社員、アルバイト、在留、カナ等）
- 条件显示规则（第 6 节表格）严格保留
- 权限矩阵（第 5 节）严格保留
- 日期统一用 `fmtDateW` 输出（格式：`2026-04-21(火)`，已包含星期）
- 名片头像：首字母 + `t.ac` 主色径向/线性渐变
- 外层不加 background，父级已有渐变背景透过来

## 10. 交付物

希望输出（桌面 + 移动端都要考虑）：
- **档案库列表**：现有表格风格过时，考虑卡片网格 or 改进表格；保留筛选区；明确公司筛选、雇佣类型筛选两层结构
- **档案详情**：
  - 重新设计顶部名片，chip 分主次层级
  - 考虑用 tab / sticky anchor nav / 折叠面板 来组织 8 个区块（尤其移动端）
  - admin-only 字段的锁定视觉要明确
  - 警告条（在留/合同过期）在合适位置能强引导
  - staff 的"个人中心"氛围与 admin 的"后台编辑"氛围可以适度区分
- **区块标题/字段样式**：统一标签 + 值 的排版
- **列表/名片的响应式断点** 规划
- 可选：微交互（hover、保存 loading、锁定字段的 tooltip）
