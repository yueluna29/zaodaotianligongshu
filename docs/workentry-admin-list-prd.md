# 工资总表（Admin 视图）UI 设计 PRD

## 1. 产品背景

- 系统：早稻田理工塾内部勤怠/工资管理系统。两家关联公司共用（世家学舍、紫陽花教育）+ 两家中国公司（早稻大连、早理金华）
- 技术栈：React 18（函数组件 hooks，**无 TypeScript**）+ Vite + Supabase
- 样式约束：**全部内联 style 对象**，从 theme `t` 取色；**禁止引入 Tailwind / CSS modules / styled-components**
- 字体：继承 body 的 `Noto Sans JP / Noto Sans SC`，**不要引入 Inter**
- 图标库：`lucide-react`
- 外层容器 ambient 渐变 blobs（蓝+青）已在其他页面使用，保持视觉延续

## 2. 使用者与场景

**使用者**：仅管理者（admin）

**核心场景**：
- 每月末处理工资前，admin 打开这个页面查看本月所有员工的工资汇总
- 从页面能一眼看出：谁本月赚了多少、各家公司的工资总盘子、异常值（工时超标、超过 28h 等）
- 点击某个员工可以钻取到该员工的工资报表详情（已有页面 `WorkEntryManager` 的单人模式）

## 3. 数据模型说明（这是给 Gemini 最重要的上下文）

系统员工分两大类，**工资算法完全不同**：

### 3.1 バイト / 外部讲师 / 兼职（Hourly，**数据完整**）

工资 = 课时费 + 交通费 + 其他报销 + 签单提成（如启用）

| 项目 | 数据来源 |
|---|---|
| 课时费 | `work_entries.work_minutes × hourly_rate`（按业务类型 `business_type` 区分时薪）|
| 交通费 | `work_entries.transport_fee` 逐条累加 |
| 其他报销 | `work_entries.other_expense` 累加 |
| 签单提成 | `commission_entries.commission_amount` 累加（仅 `employees.has_commission=true` 才有） |

### 3.2 正社員 / 契約社員 / 正社员（Full-time，**数据不全**）

工资 = 基本给 + 固定残業手当 + 固定交通费 + 签单提成 − 扣款（社保/扶养控除）

| 项目 | 数据来源 | 状态 |
|---|---|---|
| 基本给 | — | **❌ 系统未存** |
| 固定残業手当 | `employees.fixed_overtime_hours`（仅小时数） | **❌ 基准时薪未存** |
| 固定交通费 | `employees.transport_amount` | ✅ |
| 签单提成 | `commission_entries.commission_amount` | ✅ |
| 社保/扶养控除 | — | **❌ 通常线下处理** |

**UI 处理策略**（待确认）：
- 方案 A：正社一列留空或显示"—"，只汇总可算出的交通费+提成；基本给由 admin 在别处处理
- 方案 B：给 employees 表加 `base_salary` 字段，人事档案里录入后纳入汇总
- 方案 C：做一个"月度工资单"表（payslips），admin 每月手工录入正社的基本给，自动拼接其他字段

当前 PRD 假设**方案 A（正社只显示可算部分，标注"未含基本给"）**，Gemini 设计时请按这个假设。

## 4. 页面结构

### 4.1 顶部

- 页面标题：`工资总表` + 月份切换按钮（← 上月 / 当前 year月month月 / 下月 →）
- 右上角操作：`导出 CSV` 按钮（可选，后端未实现）

### 4.2 统计卡片（全公司维度）

一排 3~4 张 glass 卡片：

| 卡 | 数值 |
|---|---|
| 本月总支出 | 所有员工合计工资 `¥XX,XXX,XXX` |
| 时薪员工小计 | バイト/外部/兼职 的合计 |
| 正社员小计 | 正社/契約/正社员 可算部分（标注"未含基本给"） |
| 人数 | 有本月记录的员工数 |

### 4.3 筛选栏（两层 chip）

- 第一层（公司）：`全部 / 世家学舍 / 紫陽花教育 / 早稻大连 / 早理金华`
- 第二层（雇佣类型）：`全部 / 正社員 / 契約社員 / アルバイト / 外部講師 / 正社员 / 兼职`
- 可选：按部门筛选（教务/咨询/宣传/财务/大学院/学部/文书/语言类）

筛选后统计卡片数值也要联动更新。

### 4.4 员工汇总列表

**每一行一个员工**，建议使用卡片或分组的 table。每行包含：

| 列 | 说明 |
|---|---|
| 姓名 + 部门 chip + 雇佣类型 chip + 公司 chip | 参考人事档案的名片样式 |
| 工时数（仅 hourly 显示） | `X.X h` |
| 课时费 | `¥XXX,XXX`（仅 hourly） |
| 交通费 | `¥XXX,XXX`（hourly 累加 work_entries.transport_fee；正社显示 employees.transport_amount） |
| 其他报销 | `¥XXX,XXX` |
| 签单提成 | `¥XXX,XXX`（仅 has_commission） |
| 本月合计 | **最大字号，主色**，`¥XXX,XXX` |
| 状态 badge（可选） | 未提交 / 草稿 / 已确认 |
| 右箭头或"查看详情" | 点击跳转到该员工的工资报表详情 |

**正社员的行要有视觉提示**："未含基本给"小字 tag 标在合计旁边。

### 4.5 空状态

- 某月无任何 work_entries + commission_entries → "本月暂无工资数据"
- 筛选后空 → "当前筛选条件下无员工"

## 5. 交互细节

1. **行点击**：跳转到该员工的工资报表详情页（现有的 `WorkEntryManager` 单人模式，admin 点返回列表即可回）
2. **卡片/行 hover**：背景色 tint 变化，提示可点击
3. **筛选 chip**：多选层间叠加（公司 × 雇佣类型 都要满足）；chip 有选中态（`t.ac` 主色描边 + 浅底）
4. **月份切换**：左右箭头 + 中间可点击选择任意月（日期选择器）
5. **响应式**：
   - 桌面：列表横排表格或卡片网格
   - 移动：每行折叠成纵向卡片（姓名+chips 上，数字区下）

## 6. 视觉约束（必须遵守）

- 玻璃卡片：`rgba(255, 255, 255, 0.65)` + `backdrop-filter: blur(20px)` + 圆角 24px + 柔和阴影 `0 20px 50px -20px rgba(30, 64, 175, 0.06)`
- 主色：`t.ac = #2563EB`，hover：`t.ah = #1D4ED8`
- 状态色：绿 `t.gn = #059669` / 红 `t.rd = #DC2626` / 黄 `t.wn = #D97706`
- 文字：`t.tx` 主、`t.ts` 次、`t.tm` 弱、`t.td` disabled
- 公司色：黄（`t.wn` + `#FEF3C7` 底）
- 雇佣类型色：绿（`t.gn` + `#D1FAE5` 底）
- 部门色：主色（`t.ac` + `t.tb` 底）
- 金额用 monospace + tabular-nums，`fontVariantNumeric: "tabular-nums"`
